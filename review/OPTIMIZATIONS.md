- [x] packages/pdf2md/src/webgpu-preprocess.mjs:60 — adaptive thresholding rescans a `(2r+1)^2` neighborhood for every interior pixel — the default 262,144-pixel parity sample at radius 8 performs about 75 million luma reads per routed page and can dominate WebGPU diagnostics — compute a summed-area table or separable box sum once so each local average is O(1) (FIXED: replaced interior neighborhood rescans with summed-area lookups while preserving border, alpha, radius, and signed-bias behavior; software WebGPU adapters now retain parity evidence without claiming hardware speedup — verified by `node --test packages/pdf2md/test/webgpu-preprocess.test.mjs packages/pdf2md/test/browser-webgpu-preprocess.test.mjs`, `npm run test:api`, `npm run qa:webgpu-preprocess` with `parity=true`, and `npm run lint`; the 512x512 radius-8 sample improved from about 103 ms to about 4.4 ms)
- [x] packages/pdf2md/src/content-stream.mjs:98 — ruling-line merging uses a linear cluster search followed by repeated all-pairs merge passes — dense vector pages are O(n^2) and chained merges can approach O(n^3), making table extraction vulnerable to severe CPU growth — bucket by page/source/orientation and quantized axis coordinate, sort intervals by start, and merge each bucket in one sweep (FIXED: replaced full cluster scans with scope and coordinate buckets backed by start-ordered interval indexes, plus a lazy ordered merge-pair queue that preserves the former transitive and input-order semantics — verified by `node --test packages/pdf2md/test/content-stream.test.mjs`, a 10,000-trial seeded differential against the former algorithm, `npm run lint`, `npm run test:api`, `npm run qa:table-detection`, and `npm run corpus:run:tables`; 10,000 disjoint segments improved from about 1.14 seconds to about 23 ms)
- [ ] packages/pdf2md/src/webgpu-capability.mjs:210 — capability detection creates a real `GPUDevice` but returns only diagnostics and never destroys or reuses that device — repeated browser conversions allocate GPU resources that preprocessing cannot use and must await garbage collection — carry the probed device through an internal non-serialized execution context and dispose it deterministically, or destroy a diagnostics-only probe immediately
- [ ] packages/pdf2md/src/text-extract.mjs:83 — every content stream is independently tokenized and interpreted three times for text, ruling lines, and image draws — large PDFs pay roughly triple lexical/operator traversal cost and repeatedly allocate the full token array — expose one bounded content-stream walk that emits all three signal types (or at least reuse a single token sequence)
