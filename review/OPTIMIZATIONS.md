- [ ] scripts/qa/check-performance-regression.mjs:20 — the default throughput floor allows current performance to drop to 2 percent of baseline before the ratio check fails — severe throughput regressions can pass CI as long as the loose 10 pages-per-second absolute floor is met — set an evidence-based regression ratio per workload, typically near 0.8 to 0.9, and document any wider tolerance required for noisy CI hosts

- [ ] packages/pdf2md/src/webgpu-preprocess.mjs:60 — adaptive thresholding rescans a `(2r+1)^2` neighborhood for every interior pixel — the default 262,144-pixel parity sample at radius 8 performs about 75 million luma reads per routed page and can dominate WebGPU diagnostics — compute a summed-area table or separable box sum once so each local average is O(1)
- [ ] packages/pdf2md/src/content-stream.mjs:98 — ruling-line merging uses a linear cluster search followed by repeated all-pairs merge passes — dense vector pages are O(n^2) and chained merges can approach O(n^3), making table extraction vulnerable to severe CPU growth — bucket by page/source/orientation and quantized axis coordinate, sort intervals by start, and merge each bucket in one sweep
- [ ] packages/pdf2md/src/webgpu-capability.mjs:210 — capability detection creates a real `GPUDevice` but returns only diagnostics and never destroys or reuses that device — repeated browser conversions allocate GPU resources that preprocessing cannot use and must await garbage collection — carry the probed device through an internal non-serialized execution context and dispose it deterministically, or destroy a diagnostics-only probe immediately
- [ ] packages/pdf2md/src/text-extract.mjs:83 — every content stream is independently tokenized and interpreted three times for text, ruling lines, and image draws — large PDFs pay roughly triple lexical/operator traversal cost and repeatedly allocate the full token array — expose one bounded content-stream walk that emits all three signal types (or at least reuse a single token sequence)
